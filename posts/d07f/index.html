分治算法

分治很大程度上基于递归的，也适合用递归实现，一般分为三个过程

  1. 分解：将问题分解成一系列子问题
  2. 解决：递归求解各个子问题，若问题足够小，直接求解
  3. 合并：将子问题的结果合并成原问题

分治法适用于许多问题，特别是以下情况：

  1. 排序算法：像归并排序和快速排序这样的排序算法使用分治法。
  2. 搜索问题：二分搜索是分治法的一个典型例子。
  3. 图算法：在图遍历、最短路径和最小生成树等问题中，分治法可以用于解决一些子问题。
  4. 多项式求解：分治法可以用于多项式乘法和快速幂等问题。
  5. 并行计算：分治法适用于并行计算，因为它将问题分成多个子问题，可以在不同的处理器上并行解决。

注意：

  1. 递归停止条件：去报递归的停止条件明确，防止无线递归
  2. 子问题独立性：解决一个子问题不会影响其他子问题
  3. 子问题规模的控制：需要仔细控制子问题的规模，确保在递归中逐渐减小问题规模，否则会导致性能问题
  4. 合并步骤的有效性：确保合并步骤能高效的将子问题的解合并成原始问题的解
  5. 性能考虑：尽管分治法可以解决很多问题，但不一定是最优的解决方案

动态规划

DP是一种常用于解决优化问题的算法思想，基本原理是将复杂的问题拆分成小问题，并在解决小问题的过程中利用已经解决的子问题的结果，从而避免重复运算，提高效率，DP通常用于具有重叠子问题和最优子结构性质的问题。

动态规划的基本使用方法

1. 确定状态： 首先，需要明确定义问题的状态。状态是描述问题局部信息的一种方式，通常包括一个或多个变量，这些变量的取值描述了问题的特定情况。例如，在解决最短路径问题时，状态可以表示当前位置和已经走过的路径。

2. 定义状态转移方程： 接下来，需要确定状态之间的关系，即如何从一个状态转移到另一个状态。这通常通过递归式的状态转移方程来实现。状态转移方程描述了问题的子问题之间的关系，并指定了如何计算当前状态的最优解。

3. 初始化边界条件： 通常，动态规划问题需要一个或多个初始状态，也称为边界条件。这些边界条件是问题的基本情况，通常是已知的或者可以直接计算的。边界条件用于递归的起始点。

4. 自底向上计算或记忆化搜索： 有两种常用的动态规划解决方法。一种是自底向上的方法，从最小的状态开始逐步计算到目标状态，每个状态的计算仅依赖于之前计算的状态。另一种是记忆化搜索，通常使用一个表格或字典来存储已经计算过的状态，避免重复计算。

5. 根据问题要求获取答案： 最终，根据问题的要求从计算出的状态中获取最优解或问题的答案。

青蛙跳楼问题

一只青蛙一次可以跳上1级台阶，也可以跳上2级台阶。求该青蛙跳上一个 10 级的台阶总共有多少种跳法

假设跳到第n级台阶的跳数我们定义为f(n)，很显然就可以得出以下公式：f(n) = f(n-1) + f(n-2)，因此可以用递归来解决问题,

带备忘录的递归解法（自顶向下）

一般使用数组或者哈希map来充当备忘录

自底向上的动态规划

动态规划和带备忘录的递归解法基本思想是一致的，减少重复运算，但是空间复杂度更小

  • 带备忘录的递归，是从f(10)往f(1）方向延伸求解的，所以也称为自顶向下的解法。
  • 动态规划从较小问题的解，由交叠性质，逐步决策出较大问题的解，它是从f(1)往f(10）方向，往上推求解，所以称为自底向上的解法

动态规划有几个典型特征：最优子结构，状态转移方程，边界，重叠子问题

  • f(n-1)和f(n-2) 称为 f(n) 的最优子结构
  • f(n)= f（n-1）+f（n-2）就称为状态转移方程
  • f(1) = 1, f(2) = 2 就是边界啦
  • 比如f(10)= f(9)+f(8),f(9) = f(8) + f(7) ,f(8)就是重叠子问题。

动态规划思路

什么问题考虑用动态规划解决

一个问题可以将所有答案穷举出来，且穷举后，发现存在子重叠问题。例如：最长递增子序列、最小编辑距离、背包问题、凑零钱问题

动态规划的解题思路

动态规划的核心思想就是拆分子问题，记住过往，减少重复计算。 并且动态规划一般都是自底向上的，因此到这里，基于青蛙跳阶问题，我总结了一下我做动态规划的思路：

  • 穷举分析
  • 确定边界
  • 找出规律，确定最优子结构
  • 写出状态转移方程

  1. 穷举分析

    先自底向上穷举分析

  2. 确定边界

    通过穷举分析，我们发现，当台阶数是1的时候或者2的时候，可以明确知道青蛙跳法。f（1） =1，f(2) = 2，当台阶n>=3时，已经呈现出规律f(3) = f(2) + f(1) =3，因此f（1） =1，f(2) = 2就是青蛙跳阶的边界。

  3. 找规律，确定最优子结构

回溯算法

回溯法（back tracking）（探索与回溯法）是一种选优搜索法，又称为试探法，按选优条件向前搜索，以达到目标。但当探索到某一步时，发现原先选择并不优或达不到目标，就退回到上一步，重新选择，这种走不通就退回再走的技术为回溯法，而满足回溯条件的某个状态的点称为“回溯点”

回溯法的基本原理

  1. 选择：从候选解空间中选择一个候选解，通常是在当前可行解的基础上添加一个元素或状态
  2. 约束：检查所选择的候选解是否满足问题的约束条件，如果不满足，就放弃这个候选解
  3. 目标：检查当前候选解是否满足问题的目标。如果满足，就找到了一个解决方案，可以结束搜索，如果不满足，继续搜索下一个候选解
  4. 回退：如果无法找到合适的候选解，或者已经找到了一个解决方案，就退回到上一个状态，继续搜索

回溯法基本思想

在包含问题的所有解的解空间树中，按照深度优先搜索的策略，从根结点出发深度探索解空间树。当探索到某一结点时，要先判断该结点是否包含问题的解，如果包含，就从该结点出发继续探索下去，如果该结点不包含问题的解，则逐层向其祖先结点回溯。（其实回溯法就是对隐式图的深度优先搜索算法）。

若用回溯法求问题的所有解时，要回溯到根，且根结点的所有可行的子树都要已被搜索遍才结束。

而若使用回溯法求任一个解时，只要搜索到问题的一个解就可以结束。

全排列

给定一个无重复数字的整数数组，求其所有的排列方式。

搜索方法是，枚举每一位可能出现的情况，已经选择的数据在当前要选择的数字中不能出现，这样就可以不重不漏

说明：

  • 每一个结点表示了求解全排列问题的不同的阶段，这些阶段通过变量的「不同的值」体现，这些变量的不同的值，称之为「状态」；

  • 使用深度优先遍历有「回头」的过程，在「回头」以后， 状态变量需要设置成为和先前一样 ，因此在回到上一层结点的过程中，需要撤销上一次的选择，这个操作称之为「状态重置」；

  • 深度优先遍历，借助系统栈空间，保存所需要的状态变量，在编码中只需要注意遍历到相应的结点的时候，状态变量的值是正确的，具体的做法是：往下走一层的时候，path 变量在尾部追加，而往回走的时候，需要撤销上一次的选择(即弹出)，也是在尾部操作，因此 path 变量是一个栈；

  • 深度优先遍历通过「回溯」操作，实现了全局使用一份状态变量的效果。

八王后问题

找到八个皇后的放置方式，使得它们互相不攻击（不在同一行、同一列或同一对角线上)

从第一行开始当某一行皇后位置不与前面所有皇后位置冲突那么记录该行皇后位置并调用递归函数进入下一行，摆放下一个皇后，逐个位置摆放，若该行所有位置都被其他皇后占领，那么就回溯到上一行重新摆放上一行皇后直至所有皇后都不冲突那么记录一次方法然后回溯寻找其他摆放方法。

贪心法

贪心法（Greedy Algorithm）是一种基于局部最优选择来构建全局最优解的算法策略。在贪心法中，算法通过在每一步选择中都采取当前状态下的最佳选择，而不考虑整体的问题。这种选择可能不一定是全局最佳的，但在某些情况下，贪心法可以产生接近最优解的结果，并且具有高效性。

贪心法的基本原理如下：

  1. 建立问题模型：将问题抽象成一个数学模型，明确定义问题的目标和约束条件。
  2. 确定贪心策略：选择一种贪心策略，即在每一步中如何选择当前最优的解决方案。
  3. 迭代执行：从问题的初始状态开始，根据贪心策略，反复进行选择和执行，每一步都选择当前状态下的最佳解决方案。
  4. 检查可行性：在每一步选择后，检查所选择的解决方案是否满足问题的约束条件和目标。
  5. 结束条件：重复执行直到满足某个结束条件，通常是达到问题的最终状态或无法继续选择下去。
